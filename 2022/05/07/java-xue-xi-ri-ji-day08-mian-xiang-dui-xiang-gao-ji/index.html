类变量和类方法

类变量，可以通过类名来访问

类变量的内存布局

在JDK7和8之前静态变量是存放在方法区的静态域里面, 在JDK8以后存放在堆中, 当类加载的时候就已经随之生成了, 加载到class实例的尾部(static变量), 不管加载在哪里都不会影响静态变量的使用, 因为静态变量最核心的是会被所有的对象实例共享

共识是:

  1. 静态变量是同一个类中所有对象共享的
  2. static类变量是在类加载的时候生成的

代码块

基本介绍

基本语法

  • 下面的三个构造器都有相同的语句
  • 这样代码看起来比较冗余
  • 这时我们可以把相同的语句，放入到一个代码块中，即可
  • 这样当我们不管调用哪个构造器，创建对象，都会先调用代码块的内容
  • 代码块调用的顺序优先于构造器..

代码块使用注意事项和细节讨论

  1. static代码块也叫做静态代码块, 作用就是对类进行初始化, 而且它随着类的加载而执行, 并且只会执行一次( 类加载一次 ), 如果是普通代码块, 每创建一个对象, 就执行, 和类的加载没有任何关系。

  2. 类什么时候被加载:

    创建对象实例时(new)

    创建子类对象实例，父类也会被加载, 而且，父类先被加载，子类后被加载

    使用类的静态成员时(静态属性，静态方法), 同时使用子类的静态成员时父类也会被先加载, 会先输出父类的静态代码块

  3. 普通的代码块, 在创建对象实例时, 会被隐式的调用, 被创建一次, 就会调用一次。如果只是使用类的静态成员时, 普通代码块并不会执行

  4. 创建一对象时, 在一个类调用顺序是:

    调用静态代码块和静态属性初始化(注意: 静态代码块和静态初始化调用的优先级相同, 如果有多个静态代码块和多个静态变量初始化, 则按照他们定义的顺序调用)

    调用普通代码块和普通属性的初始化( 注意: 普通代码块和普通属性初始化调用的优先级一样,如果有多个普通代码块和多个普通属性初始化, 则按照定义顺序调用)

    在整个对象创建的过程中构造器的调用是在最后的

  5. 构造器的最前面其实隐含了super() 和调用普通代码块, 静态相关的代码块, 属性初始化, 在类加载时, 就执行完毕

  1. 创建一个子类对象时( 继承关系 ),被调用的顺序是:

    (1) 父类的静态代码块和静态属性( 优先级一样, 按定义顺序执行 )

    (2) 子类的静态代码块和静态属性( 优先级一样, 按定义顺序执行 )

    (3) 父类的普通代码块和普通属性初始化( 优先级一样, 按定义顺序执行 )

    (4) 父类的构造方法

    (5) 子类的普通代码块和普通属性初始化( 优先级一样, 按定义顺序执行 )

    (6) 子类的构造方法

  2. 静态代码块只能调用静态成员, 普通代码块可以调用任意成员

补充: 没有继承关系的构造器执行super()调用的是object类的无参构造器, 不会输出结果

单例设计模式

单例模式应用实例

  1. 步骤 [单例模式-饿汉式] (可能还没去用这个对象, 但是已经加载好了所以饿汉式)

    饿汉式可能造成资源的浪费

  • 将构造器私有化
  • 在类的内部直接创建对象(该对象是 static)
  • 提供一个公共的 static 方法，返回 gf 对象
  1. 步骤 [单例模式-懒汉式]
  • 仍然构造器私有化
  • 定义一个static静态属性对象
  • 提供一个 public 的 static 方法，可以返回一個 Cat 对象
  • 懒汉式，只有当用戶使用 getInstance 时，才返回 cat 对象, 后面再次调用时，会返回上次创建的 cat 对象, 从而保证了单例

抽象类

  • 父类方法不确定性的问题
  • 考虑将该方法设计为抽象(abstract)方法
  • 所谓抽象方法就是没有实现的方法
  • 所谓没有实现就是指，没有方法体
  • 当一个类中存在抽象方法时，需要将该类声明为 abstract 类
  • 一般来说，抽象类会被继承，有其子类来实现抽象方法.

抽象类的介绍

抽象类使用的注意事项和细节讨论

  • 抽象类，不能被实例化
  • 抽象类不一定要包含 abstract 方法。也就是说,抽象类可以没有 abstract 方法
  • 一旦类包含了 abstract 方法,则这个类必须声明为 abstract
  • abstract 只能修饰类和方法，不能修饰属性和其它的
  • 抽象类的本质还是类，所以可以有类的各种成员
  • 抽象方法不能有主体, 也就是方法体
  • 如果一个类继承了抽象类，则它必须实现(从语法上看只要有了方法体就是实现, 不关心其中具体写了什么)抽象类的所有抽象方法，除非它自己也声明为 abstract 类
  • 抽象方法不能使用 private、final 和 static 来修饰，因为这些关键字都是和重写相违背的

final和abstract，private和abstract，static和abstract，这些是不能放在一起的修饰符

因为abstract修饰的方法是必须在其子类中实现（覆盖），才能以多态方式调用，以上修饰符在修饰方法时期子类都覆盖不了这个方法，final是不可以覆盖，private是不能够继承到子类，所以也就不能覆盖，static是可以覆盖的，但是在调用时会调用编译时类型的方法，因为调用的是父类的方法，而父类的方法又是抽象的方法，又不能够调用，所以上的修饰符不能放在一起。abstract用在类或者方法上。说明要建立子类来实现。static用在方法上，是不需要子类来实现，不抽象为何要子类来实现。二者用在一起，就自相矛盾了。

抽象类最佳实践-模板设计模式

基本介绍

模板设计模式也就是利用抽象类将每个子类一样的方法写在父类里面,同时将需要每个子类分别去实现的方法设计为抽象类, 然后在子类中分别去实现

接口

USB插槽就是现实中的接口

接口里面默认的都是抽象方法和常量

规范并且可以统一调用

注意事项和细节

  • 接口不能被实例化

  • 接口中所有的方法是 public 方法, 接口中抽象方法，可以不用 abstract 修饰

    void say()实际上是abstract void say()

  • 一个普通类实现接口,就必须将该接口的所有方法都实现,可以使用 alt+enter 来解决

  • 抽象类去实现接口时，可以不实现接口的抽象方法

  • 一个类同时可以实现多个接口

  • 接口中的属性,只能是 final 的，而且是 public static final 修饰符

  • 接口中属性的访问形式: 接口名.属性名

  • 接口不能继承其它的类,但是可以继承多个别的接口, 接口和接口只能是继承的关系

  • 接口的修饰符 只能是 public 和默认，这点和类的修饰符是一样的

实现接口 vs 继承类

  • 当子类继承了父类，就自动的拥有父类的功能
  • 如果子类需要扩展功能，可以通过 实现接口的方式扩展.
  • 可以理解 实现接口 是 对 java 单继承机制的一种补充.

即: 接口规范性 + 动态绑定机制(后面集合源码有所体现)

接口的多态

只要实现了UsbInterface接口的类的对象实例都可以传递进去作为形参

多态参数: 在前面UsbInterface中, 即可以接受手机对象, 又可以接受相机对象, 就体现了接口的多态特性 ( 接口类型的变量if01可以指向实现了IF接口的类的对象实例 )

多态数组: 接口的多态数组可以存放所有实现了接口方法的类的对象实例

内部类(重要)

基本介绍

内部类的分类

局部内部类

  • 局部内部类是定义在外部类的局部位置,通常在方法中

  • 局部内部类的本质仍然是一个类(类相关的内容对局部内部类仍然适用)

  • 可以直接访问外部类的所有成员，包含私有的

  • 不能添加访问修饰符,但是可以使用 final 修饰

  • 作用域 : 仅仅在定义它的方法或代码块中

  • 局部内部类可以直接访问外部类的成员

  • 外部类在方法中，可以创建 Inner02 对象，然后调用方法即可(必须在作用域内)

  • 外部其他类不能访问局部内部类(因为局部内部类还是一个局部变量)

  • 如果外部类和局部内部类的成员重名时，默认遵循就近原则，如果想访问外部类的成员，

    使用 外部类名.this.成员 去访问

匿名内部类的使用注意事项

  1. 匿名内部类本身还是继承了Person的, 匿名内部类既是一个类的定义,同时本身也是一个对象, 因此它既有类的特征也有创建对象的特征

  2. 可以直接访问外部类的所有成员, 包含私有的

  3. 不能添加访问修饰符, 因为他的地位就是一个局部变量

  4. 作用域: 仅仅在定义它的方法或代码块中

  5. 匿名内部类可以直接访问外部类成员

  6. 外部类不能访问匿名内部类(还是由于地位就是一个局部变量)

  7. 如果外部类和局部内部类的成员重名时，默认遵循就近原则，如果想访问外部类的成员，

    使用 外部类名.this.成员 去访问

匿名内部类最佳实践

当做实参直接传递，简洁高效

成员内部类的使用

  • 成员内部类是定义在外部类的成员位置, 并且没有static修饰, 并不用定义在类的方法中
  • 可以直接访问外部类的所有成员
  • 可以添加任意修饰符, 成员内部类本质上就是一个成员
  • 在外部类创建对象, 然后调用内部类的方法
  • 成员内部类访问外部类, 直接访问
  • 外部其他类访问成员内部类三种方式

  • 如果外部类和局部内部类的成员重名时，默认遵循就近原则，如果想访问外部类的成员，

    使用 外部类名.this.成员 去访问

静态内部类的使用

  • 放在外部类的成员位置
  • 使用 static 修饰
  • 可以直接访问外部类的所有静态成员，包含私有的，但不能直接访问非静态成员
  • 可以添加任意访问修饰符(public、protected 、默认、private),因为它的地位就是一个成员
  • 作用域 ：同其他的成员，为整个类体
  • 外部类—访问——>静态内部类 访问方式：创建对象，再访问
  • 静态内部类 —访问——>外部类 访问方式：直接访问
  • 外部其他类—访问——>静态内部类 访问方式：创建对象，再访问

  • 如果外部类和局部内部类的成员重名时，默认遵循就近原则，如果想访问外部类的成员，

    使用 外部类名.成员 去访问
